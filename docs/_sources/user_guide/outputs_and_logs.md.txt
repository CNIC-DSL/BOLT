# Outputs and Log Navigation

This page explains where to find logs, aggregated results, and generated artifacts after a run finishes, and provides a recommended troubleshooting order for common issues.

## 1. Key Output Directories (Under the Working Directory `--output-dir`)

- `logs/`  
  Runtime logs for each combo, including per-stage logs. Check here first when debugging errors.

- `results/`  
  Directory for summary files and per-method results. It typically contains:
  - `summary_{result_file}.csv`

- `outputs/`  
  Training/evaluation artifacts (models, predictions, caches, etc.), organized by method and combo.

Note on `--init-only`:
- If you run with `--init-only`, the directory skeleton (and linked directories) will be created, but it is normal that there are **no combo stage logs** and **no new summary records**, because no combos are executed.

## 2. Log Directory Organization

The log directory for each combo is typically organized as follows:

logs/{task}/{method}/{dataset}/
  kr{known_cls_ratio}/
  lr{labeled_ratio}/
  {fold_type}_{fold_num}_{fold_idx}/
  seed{seed}/

This directory usually contains:
- `stage1.log`, `stage2.log`, ...: stdout/stderr for each stage

At the beginning of each stage log, you will typically see two pieces of information:
- `CMD`: the actual command executed
- `ARGS_JSON`: the full set of parameters for this combo (useful for reproduction and debugging)

## 3. Result Files and Summary Files

### 3.1 Per-Method Result File (`results.csv`)

By default, the framework reads per-method results from:
results/{task}/{method}/results.csv

The framework takes the last row of this CSV as the “latest result record” and writes it into the summary table.

### 3.2 Summary File (`summary_{result_file}.csv`)

The summary file is located at:
results/summary_{result_file}.csv

Whenever a combo successfully collects results, the summary table appends one new row.

Note on `--init-only`:
- `--init-only` does not run stages and does not write method results, so the summary table will not append new rows.

## 4. Recommended Troubleshooting Order

Issue A: A combo fails / runtime error occurs
1) Locate the corresponding `stageX.log` for that combo
2) Check the error stack trace and key parameters (`ARGS_JSON`)
3) Use `CMD` to reproduce the single run directly for debugging

Issue B: The run finishes but the summary has no new records
1) Check whether any stage return code is non-zero (on failure, the pipeline stops and will not write to the summary)
2) Check whether `results/{task}/{method}/results.csv` exists and is non-empty
3) Check whether the columns written by the method are complete (especially metric columns and the `args` field)

Issue B (init-only): You used `--init-only`
- This is expected: `--init-only` only initializes the workspace and does not execute combos, so it will not produce `results.csv` or append the summary.

Issue C: The output directory structure is not as expected
1) Confirm that `--output-dir` points to the experiment root directory (do not point it directly to an `outputs` subdirectory)
2) Confirm that `paths.results_dir` / `paths.logs_dir` are relative paths (relative paths will be created under the working directory)
