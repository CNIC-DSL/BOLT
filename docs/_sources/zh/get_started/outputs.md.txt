# 输出目录结构与说明

本节介绍 **bolt-lab** 运行时输出的写入位置、实验完成后通常会创建哪些目录，以及推荐的使用方式，帮助你快速定位日志和结果。

## 1. 工作目录（`--output-dir`）的含义

运行时，你通过 `--output-dir` 指定一个**工作目录**。框架会在该目录下创建当前实验的运行环境和输出文件夹，并将**日志、汇总结果和训练/评估产物**写入其中。这使得迁移、打包和复现运行变得方便。

建议将 `--output-dir` 指向一个专用的实验根目录（例如 `~/tmp/exp_001`），而不是直接指向 `outputs/` 或 `logs/` 等子目录。

## 2. 运行后通常创建的目录

以 `--output-dir ~/tmp` 为例，运行后你将看到类似如下的结构（示例）：

~/tmp/
  outputs/            训练/评估产物（模型、预测结果、缓存等）
  results/            按方法汇总的结果和汇总文件
  logs/               运行时日志（用于调试错误和跟踪阶段进度）
  configs/            本次运行使用的 YAML 配置文件副本
  data -> ...         指向 --data-dir 的符号链接（或包内置的 _data 目录）
  pretrained_models -> ...  指向 --model-dir 的符号链接

注意：
- 请首先关注 `logs/`、`results/` 和 `outputs/`。
- `configs/` 是一个真实目录，其中的 YAML 配置文件从包中复制而来，保留了本次运行的确切配置状态。
- `data` 和 `pretrained_models` 是符号链接，分别指向你通过 `--data-dir` 和 `--model-dir` 指定的目录。

## 3. 推荐用法

### 3.1 如何修改配置文件

建议**不要**直接编辑工作目录下 `configs/` 中的文件作为长期的配置来源。

更稳妥的做法是：
- 将示例配置复制到你自己的目录并在那里修改。
- 运行时通过 `--config` 传入该配置文件的绝对路径。

这样可以避免不同实验之间的意外覆盖，也更便于版本控制。

### 3.2 排查问题时优先查看的位置

- 运行时错误、卡住或阶段失败：请先检查 `logs/`。通常可以在其中找到错误堆栈信息和失败的阶段。
- 实验汇总结果：请检查 `results/`（通常包含按方法分类的结果文件和汇总 CSV）。
- 模型和预测产物：请检查 `outputs/`。

## 4. 常见问题及避免方法

### 4.1 避免输出目录嵌套

确保 `--output-dir` 是实验根目录。
不要将 `--output-dir` 设为类似 `.../outputs` 的路径，否则可能产生 `outputs/outputs` 这样的嵌套结构，给查找和清理带来困难。

### 4.2 关于 `--model-dir`（补充说明）

`--model-dir` 用于指定模型/缓存目录（用于读写预训练模型及相关缓存）。
工作目录下的 `pretrained_models/` 目录是该入口点的容器。建议将 `--model-dir` 设为一个稳定的、可读写的位置，以减少重复下载和准备的成本。
