# 指标指南

本页面说明汇总表中常见指标的含义以及如何解读它们。不同任务使用不同的指标字段；以下描述最常见的约定。

## 1. 汇总表字段结构

`summary_{result_file}.csv` 通常包含两类字段：
- 实验设置字段：`method`、`dataset`、`known_cls_ratio`、`labeled_ratio`、`cluster_num_factor`、`seed`（以及可能的 fold 信息和额外超参）
- 指标字段：因任务而异（例如 `ACC`、`H-Score`、`F1` 等）
- `args`：一个 JSON 字符串，记录关键运行参数（用于可复现性和去重）

推荐的解读方式：
- 横向对比：固定 `dataset` / ratios / folds，比较不同方法之间的指标差异
- 纵向对比：固定 `method`，比较不同 ratios / seeds / folds 下的稳定性和趋势

## 2. 常见 GCD 指标（示例）

- `ACC`
  整体聚类/分类准确率（具体定义取决于方法实现，通常反映整体分配质量）。

- `H-Score`
  一个综合指标，旨在平衡已知类和新类的性能；常用作整体评估分数。

- `K-ACC`
  已知类准确率。

- `N-ACC`
  新类准确率。

- `ARI`（Adjusted Rand Index，调整兰德指数）
  衡量聚类结果与真实标签之间的一致性，已对随机一致性进行校正。

- `NMI`（Normalized Mutual Information，归一化互信息）
  衡量聚类与真实标签之间的互信息一致性，取值通常在 [0, 1] 范围内。

解读建议：
- 如果某方法 `K-ACC` 高但 `N-ACC` 低，通常表明其新类发现能力较弱。
- `H-Score` 通常更适合用于整体排名，但仍应结合分解后的 K/N 指标来理解具体原因。

## 3. 常见 OpenSet 指标（示例）

- `F1`
  整体 F1 分数（精确率和召回率的调和平均数），用作通用评估指标。

- `K-F1`
  已知类的 F1 分数。

- `N-F1`
  新类/未知类的 F1 分数。

解读建议：
- `K-F1` 和 `N-F1` 之间通常存在权衡关系。
- 仅看整体 `F1` 可能会掩盖未知类识别的弱点，因此建议同时查看 `K-F1` 和 `N-F1`。

## 4. 与方法实现的关系

框架不强制要求特定的指标列 -- 它们基于约定，由各方法脚本自行写入。不同方法可能会在 `results.csv` 中写入额外的列（例如 loss、运行时间、额外超参等）。
汇总时，框架通常保留核心字段并追加 `args` 以确保可追溯性。
如果您需要额外的指标出现在汇总表中，建议在方法侧统一结果列名，并确保它们被写入 `results/{task}/{method}/results.csv`。
